Dokumentation aus XOEV-Bibliothek:
 
Mit diesem XML-Attribut wird die Kennung der Codeliste bermittelt, in deren Kontext der jeweilige Code zu interpretieren ist. Die Kennung identifiziert die Codeliste, nicht jedoch deren Version eindeutig.

Wird bereits im Rahmen des XV-Standards eine Kennung vorgegeben (es handelt sich in diesem Fall um einen Code-Typ 1, 2 oder 3) darf auf eine nochmalige Angabe der Kennung bei der bermittlung eines Codes verzichtet werden. Aus diesem Grund ist das XML-Attribut listURI zunchst als optional deklariert.
listVersionID xs:normalizedString 2.0 required 
Dokumentation aus XOEV-Bibliothek:
 
Die konkrete Version der zu nutzenden Codeliste wird mit diesem XML-Attribut bertragen.

Analog zum listURI ist die Bestimmung der Version einer Codeliste bei der bertragung eines Codes zwingend. Die Version kann jedoch ebenfalls bereits im XV-Standard festgelegt werden (es handelt sich in diesem Fall um einen Code-Typ 1 oder 2).
